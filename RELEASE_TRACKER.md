# 🎮 RNGames — Release Tracker

Each game is an independent app variant built and submitted separately via EAS Build.
Update the checkboxes as you progress through each stage.

---

## How to Build & Submit a Game

### Step 1 — Build
```bash
# iOS
VARIANT=<variant-key> eas build --platform ios --profile production

# Android
VARIANT=<variant-key> eas build --platform android --profile production
```

### Step 2 — Submit
```bash
# iOS → App Store Connect
VARIANT=<variant-key> eas submit --platform ios --latest

# Android → Google Play
VARIANT=<variant-key> eas submit --platform android --latest
```

### Step 3 — Release
- **iOS:** App Store Connect → TestFlight testing → Submit for Review → Release
- **Android:** Google Play Console → Internal Testing → Closed Testing → Production → Release

## Pre-Release Checklist (Do Once, Applies to All Games)

- [ ] Apple Developer account active and paid
- [ ] Google Play Developer account active and paid
- [ ] EAS account linked (`eas login`)
- [ ] `eas.json` production profiles configured
- [ ] Apple Distribution certificate created in EAS
- [ ] Google Play service account JSON added to EAS secrets
- [ ] App icons generated for all sizes (use `npx expo-asset-utils` or Figma)
- [ ] Screenshots prepared: iPhone 6.5", iPhone 5.5", iPad 12.9" (iOS); phone + 7" tablet (Android)
- [ ] Privacy Policy URL ready (required by both stores)
- [ ] Age rating / content rating completed

## Useful Commands

```bash
# List all EAS builds
eas build:list

# Check build status
eas build:view <build-id>

# Run a specific variant locally (web preview)
VARIANT=snake npx expo start --web

# Run all variants build in sequence (example shell loop)
for VARIANT in snake circle-shrink laser-dodge pulse-lanes math-rush \
  gravity-flip color-clash stack-blocks simon-says number-order \
  tap-rhythm brick-breaker slice-frenzy tile-shift color-flood; do
  echo "Building $VARIANT..."
  VARIANT=$VARIANT eas build --platform all --profile production --non-interactive
done
```
